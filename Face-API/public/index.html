EZ Face API

Welcome to the EZ Face API, powered by the Cognitive Services API from Azure. This GraphQL API allows you to easily detect faces found in requested image URLs, or create and manage your own FaceLists. You can even Find Similar faces in your facelists by comparing the list to a new face found in a requested image URL.

GraphQL enables you to quickly request exactly what you want to retrieve from the API. This is useful because it is likely that you are only interested in specific features of a DetectedFace, like it's Happiness, Rectangle, or Hair Color.

To get started with the API, I suggest you first explore the detect faces endpoint. Open the GraphQL Playground at /graphiql for this endpoint and explore what face attributes you can find. The response documentation can either be read here (on the /docs documentation page), or in the playground itself by clicking on "DOCS" on the right. When in the Playground, assign query parameters like "url" in the Query Variables tab at the bottom, by setting the variable to a value like this: {"url":"enter_image_url_here"}.

Try the detect faces endpoint now!

Then, you can also use facelists to find similar faces. To work with this functionality, first create a facelist. Then, add a new face to the facelist by submitting an image url with a single face, along with the unique facelistID you assigned to your new facelist. Add as many faces as you want. Finally, find similar faces by comparing a new face to your facelist.

Queries

faces

Given the url of an image, return all of the detected faces in the image. These faces and their given faceIDs and attributes are temporary. To save a permanent face (no image is saved), create a FaceList and add a face to it.

Images should be JPEG, PNG, GIF (first frame), or BMP format.

For most accurate results, images should be oriented so that a face's eyes are at the top and the chin is at the bottom.

Faces that are between 36x36 pixels and 4096x4096 pixels can be detected, provided that the total image size is proportional. For example, a minimum face size of 36x36 can be detected in a 1920x1080px image, but larger images will have a larger minimum face size.

Faces are less detectable if:

  • the image has extreme lighting
  • the face (and/or eyes) is obstructed
  • the face has unusual hair type or facial hair
  • the face has extreme expressions
Returns [DetectedFace!]!

Name Description
url - String! A URL of an image with 1 or more detectable faces

Example

Query
query faces($url: String!) {
  faces(url: $url) {
    FaceID
    FaceRectangle {
      ...RectangleFragment
    }
    FaceLandmarks {
      ...LandmarksFragment
    }
    FaceAttributes {
      ...AttributesFragment
    }
  }
}
Variables
{"url": "xyz789"}
Response 
{
  "data": {
    "faces": [
      {
        "FaceID": "abc123",
        "FaceRectangle": Rectangle,
        "FaceLandmarks": Landmarks,
        "FaceAttributes": Attributes
      }
    ]
  }
}

Queries

findSimilarFace

FindSimilarFace searches the provided FaceList for faces that are similar to the provided ProbeFace.

The URL should be an image with only one face. The face will be detected and used to find a similar face in the facelist. Attributes of the detected face will be returned.

Returns a SimilarFaceResponse!

Name Description
ProbeFaceURL - String The URL of a face to probe for a similar match.
FaceListID - String! The FaceList gallery of faces to find a similar match to the probe face. The FaceListID should be a user created ID using the createFaceList() mutation.

Example

Query
query findSimilarFace($ProbeFaceURL: String, $FaceListID: String!) {
  findSimilarFace(ProbeFaceURL: $ProbeFaceURL, FaceListID: $FaceListID) {
    DetectedFace {
      ...DetectedFaceFragment
    }
    FaceList {
      ...FaceListFragment
    }
    SimilarFaces {
      ...SimilarFaceFragment
    }
  }
}
Variables
{"ProbeFaceURL": "abc123", "FaceListID": "abc123"}
Response 
{
  "data": {
    "findSimilarFace": {
      "DetectedFace": DetectedFace,
      "FaceList": FaceList,
      "SimilarFaces": [SimilarFace]
    }
  }
}

Mutations

addFaceToList

Add a face detected from the provided url to the list with the provided FaceListID.

Returned: FaceID & FaceList FaceID is the generated unique identifier of the detected and saved Face. You should likely store this value with a name or other information so that you can refer back to what this face represents or where it is from.

FaceList is the resulting FaceList the face was saved to.

There should only be one face detected in the image.

Returns a FaceResponse!

Name Description
URL - String! URL to an image that contains a single detectable face to be added to the list
FaceListID - String! The user-specified ID that uniquely identifies this FaceList.

Example

Query
mutation addFaceToList($URL: String!, $FaceListID: String!) {
  addFaceToList(URL: $URL, FaceListID: $FaceListID) {
    FaceList {
      ...FaceListFragment
    }
    FaceID
  }
}
Variables
{"URL": "abc123", "FaceListID": "xyz789"}
Response 
{
  "data": {
    "addFaceToList": {
      "FaceList": FaceList,
      "FaceID": "xyz789"
    }
  }
}

Types

Attributes

Attributes of the detected face

Field Name Description
Age - Float Estimated age of the detected face in years.
Gender - String Predicted gender of the detected face is "female" or "male".
Smile - Float Estimated smile intensity of the detected face where 0 is no smile and 1 is a strong smile.
Moustache - Float Estimated length of the moustache on the detected face where 0 is no moustache and 1 is a very long moustache.
Beard - Float Estimated length of the beard on the detected face where 0 is no beard and 1 is a very long beard.
Sideburns - Float Estimated length of the sideburns on the detected face where 0 is no sideburns and 1 is very long sideburns.
Glasses - String Predicted type of glasses on the detected face. Values could be "NoGlasses", "ReadingGlasses", "Sunglasses", or "SwimmingGoggles".
Emotion - Emotion The intensity of each type of emotion that can be detected on the face. Values range between [0,1] where 0 is no emotion and 1 is an intense emotion.
HairColor - String

Highest confidence hair color of the detected face. If the color cannot be detected, the hair is "invisible", or the individual is likely "bald" then the returned color is "Unknown".

A detected hair color could be "black", "blond", "brown", "gray", "red", "white", or "other".
EyeMakeup - Boolean EyeMakeup predicts if the detected face is wearing eye makeup (true) or not (false).
LipMakeup - Boolean LipMakeup predicts if the detected face is wearing lipstick (true) or not (false).
Accessories - [String] Accessories around the detected face, including 'headwear', 'glasses' and 'mask'. Empty array means no accessories detected. Note this is after a face is detected. Large mask could result in no face to be detected.
Blur - Float Blur is a value between [0,1] that indicates the blurriness of a detected face where 0 is not blurry and 1 is very blurry.
Exposure - Float Exposure is a value between [0,1] that indicates the exposure of a detected face where values close to 0 indicate underexposure and values closer to 1 indicate overexposure. Values close to .5 indicate good exposure.
Noise - Float Noise indicates how noisy the detected face image is where 0 is no noise and 1 is very noisy.
Example
{
  "Age": 987.65,
  "Gender": "abc123",
  "Smile": 987.65,
  "Moustache": 123.45,
  "Beard": 123.45,
  "Sideburns": 987.65,
  "Glasses": "xyz789",
  "Emotion": Emotion,
  "HairColor": "xyz789",
  "EyeMakeup": false,
  "LipMakeup": false,
  "Accessories": ["abc123"],
  "Blur": 123.45,
  "Exposure": 123.45,
  "Noise": 987.65
}

Types

Rectangle

Rectangle gives the top left corner of a detected face and how long (height) and wide (length) the detected face is.

To draw a rectangle around the detected face:

Top-left corner: (Top, Left)

Top-right corner: (Top, Left+Width)

Bottom-left corner: (Top+Height, Left)

Bottom-right corner: (Top+Height, Left+Width)

Field Name Description
Width - Int! Width along X-axis of the detected face
Height - Int! Height along Y-axis of the detected face
Top - Int! Y-intercept of the top of the detected face
Left - Int! X-intercept of the left side of the detected face
Example
{"Width": 123, "Height": 987, "Top": 987, "Left": 987}
